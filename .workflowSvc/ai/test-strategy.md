# Test Strategy

## 테스트 원칙

- 구현보다 테스트 먼저 작성 (TDD 권장)
- 외부 의존성은 반드시 Mock 처리
- 테스트 커버리지보다 핵심 경로 우선

## 테스트 계층

| 계층 | 대상 | 도구 |
|------|------|------|
| Unit | 함수, 클래스 단위 | 프로젝트 설정에 따름 |
| Integration | 모듈 간 연동 | 프로젝트 설정에 따름 |
| E2E | 전체 사용자 흐름 | 프로젝트 설정에 따름 |

## AI 테스트 작성 규칙

- plan.md에 명시된 기능에 대해서만 테스트 작성
- 기존 테스트를 삭제하거나 약화(skip, any)하지 않음
- 테스트 파일 위치는 프로젝트 컨벤션 따름
- 테스트명은 `[대상] should [동작] when [조건]` 형식 권장

## 완료 기준 연계
