# Process Log

작업이 완료된 후 사람이 읽는 진행 이력입니다.
스프린트/버전 단위로 어떤 작업이 이루어졌는지 요약합니다.

> AI가 작성하지 않습니다. 사람이 직접 정리하거나 AI에게 요청하여 작성합니다.
> 세부 커밋 이력은 git log, 버전별 변경사항은 .workflowSvc/docs/changelog.md를 참조합니다.

## 기록 형식

```markdown
## [날짜 또는 버전] [작업 제목]

### 진행된 이슈
- ISSUE-XXX: [제목] — 완료
- ISSUE-XXX: [제목] — 완료

### 주요 결정
- DECISION-XXX: [결정 내용 요약]

### 산출물
- 변경된 주요 파일 또는 기능 목록

### 특이사항
- 버그, 블로커, 다음 작업으로 넘어간 사항 등
```
