### 🟡 중간 (다음 스프린트 권장)

### 🟢 낮음 (여유 있을 때)

<!-- 문서화, 스타일 통일, 리팩터링 등 -->

기록 기준:

- 즉시 장애로 이어지지 않는 품질 개선
- 문서 정리, 개발 경험 개선, 경미한 리팩터링

## 우선순위 판단 기준

- 사용자 영향도: 실제 사용자 기능 장애 여부
- 운영 영향도: 장애 대응 시간, 모니터링 난이도
- 변경 비용: 구현/검증/배포 복잡도
- 재발 가능성: 동일 유형 문제 반복 가능성
- 선행 의존성: 다른 작업을 막는지 여부

## 실행 방법

개선사항 중 진행할 항목을 `.workflowSvc/workflow/1.brief.md`에 옮겨 적으면 AI가 plan.md 이슈로 변환합니다.

```text
예시) .workflowSvc/workflow/1.brief.md에 아래처럼 작성:
- 인증 토큰 만료 처리가 없어. 개선 필요.
- DB 쿼리에 인덱스가 없어서 느림. 최적화해줘.
```

실행 팁:

- 개선 항목은 한 번에 1~2개만 brief로 이동
- 각 항목에 "증상 + 기대 결과"를 함께 적으면 plan 품질이 높아짐
