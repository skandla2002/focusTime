  ```cmd
  choco install temurin21
  ```

  ```cmd
  java -version
  ```

  - URL: `https://developer.android.com/studio#command-tools`
  - 페이지 하단 **"Command line tools only"** > Windows용 zip 다운로드

  ```powershell
  mkdir C:\Android\cmdline-tools\latest
  Expand-Archive "$env:USERPROFILE\Downloads\commandlinetools-win-*.zip" -DestinationPath "C:\temp-android"
  ```

### ISSUE-030: 집중↔휴식 자동 전환 시 타이머 연속 진행

> 출처: "집중 25분에서 휴식 5분으로 이동 시 시간이 초기화되지 않고 계속 진행되어야 함"

- 배경:
  - 현재 `tick()`, `onForeground()` 에서 타이머 완료 시 다음 모드를 `status: 'idle'`로 전환
  - 사용자가 원하는 동작: 자동 전환 시 다음 모드 타이머가 즉시 자동 시작
  - 수동 탭 클릭(`switchMode`)은 기존대로 idle 유지

- 구현 내용:

  파일: `src/store/timerStore.ts`

  **`tick()` 수정** — 타이머 완료 시 다음 모드를 `status: 'running'`으로 자동 시작:

  ```ts
  // 변경 전
  set({ mode: nextMode, status: 'idle', timeLeft: getDurationForMode(nextMode), ... })

  // 변경 후
  set({ mode: nextMode, status: 'running', timeLeft: getDurationForMode(nextMode), sessionStart: Date.now(), ... })
  ```

  **`onForeground()` 동일 수정** — 백그라운드에서 복귀 시 다음 모드도 자동 시작

  > `switchMode()`는 수정하지 않음 — 수동 탭 전환은 idle 유지가 자연스러움

- 수정 대상 파일:
  - [ ] `src/store/timerStore.ts` — `tick()`, `onForeground()` 완료 후 `status: 'running'` 자동 시작
  - [ ] `src/store/timerStore.test.ts` — 자동 전환 후 running 상태 테스트 케이스 추가
- 테스트 필요: Yes — 집중 타이머 완료 후 휴식 타이머 자동 시작 확인
- 완료 기준:
  - 집중 25분 완료 → 휴식 5분 타이머 자동 시작 (idle 없이 바로 running)
  - 휴식 5분 완료 → 집중 25분 타이머 자동 시작
