### ISSUE-025: Android App Bundle 릴리즈 서명 및 빌드 자동화

- 구현 내용 (환경 세팅 — 개발 PC에서 1회만 수행):

  **Step 0: JDK 21 설치 (필수 — Capacitor 6.x 요구사항)**

  - **Temurin 21 (LTS)** > Windows x64 > `.msi` 다운로드

  방법 B — Chocolatey:

  ```cmd
  choco install temurin21
  ```

  설치 확인 (새 CMD 창):

  ```cmd
  java -version
  ```

  `openjdk version "21.x.x"` 출력되면 성공

  > ⚠ JDK 17이 이미 설치되어 있다면 `JAVA_HOME`을 JDK 21 경로로 변경해야 함:
  >
  > ```cmd
  > setx JAVA_HOME "C:\Program Files\Eclipse Adoptium\jdk-21.x.x.x-hotspot" /M
  > ```

  **Step 1: Android Command Line Tools 다운로드**
  - URL: `https://developer.android.com/studio#command-tools`
  - 페이지 하단 **"Command line tools only"** > Windows용 zip 다운로드

  **Step 2: 압축 해제 및 폴더 구조 생성**

  ```powershell
  mkdir C:\Android\cmdline-tools\latest
  Expand-Archive "$env:USERPROFILE\Downloads\commandlinetools-win-*.zip" -DestinationPath "C:\temp-android"
  Move-Item "C:\temp-android\cmdline-tools\*" "C:\Android\cmdline-tools\latest\"
  ```

  최종 구조: `C:\Android\cmdline-tools\latest\bin\sdkmanager.bat`

  **Step 3: 필수 SDK 컴포넌트 설치**

  ```powershell
  $env:ANDROID_HOME = "C:\Android"
  C:\Android\cmdline-tools\latest\bin\sdkmanager.bat --sdk_root="C:\Android" "platform-tools" "build-tools;34.0.0" "platforms;android-34"
  ```

  - 라이선스 동의 프롬프트에서 `y` 입력

  **Step 4: 시스템 환경변수 설정 (CMD — 관리자 권한)**

  ```cmd
  setx ANDROID_HOME "C:\Android" /M
  setx Path "%Path%;C:\Android\platform-tools" /M
  ```

  - `/M` 플래그 = 시스템 전체(Machine) 적용
  - 설정 후 **CMD 창을 닫고 새로 열어야** 적용됨
  - 확인: `echo %ANDROID_HOME%` → `C:\Android` 출력되면 성공

  - 파일 위치: `android/local.properties`
  - 내용:

    ```properties
    sdk.dir=C\:\\Android
    ```

  - ⚠ Windows 경로는 역슬래시(`\`)를 이스케이프하여 `C\:\\Android` 형식으로 작성

  - `app-release.aab`(미서명)와 `app-release-signed.aab`(서명완료) 두 파일 생성 — **Play Console에는 signed 파일 업로드**

### ISSUE-037: 화면 꺼짐 방지 (Wake Lock)

> 출처: 타이머 실행 중 화면이 꺼지면 타이머 진행 여부 확인 불가 — 실행 중 화면 ON 유지 필요

- 배경:
  - 타이머 `status === 'running'` 중에는 화면이 자동으로 꺼지지 않아야 함
  - 타이머 정지·완료 시에는 Wake Lock 해제하여 배터리 절약
  - Web API `navigator.wakeLock` 사용 (Capacitor 네이티브 플러그인 불필요)

- 구현 내용:

  **`src/screens/TimerScreen.tsx`에 Wake Lock 추가**

  ```ts
  import { useEffect, useRef } from 'react'

  const wakeLockRef = useRef<WakeLockSentinel | null>(null)

  useEffect(() => {
    if (status === 'running' && 'wakeLock' in navigator) {
      navigator.wakeLock.request('screen')
        .then((lock) => { wakeLockRef.current = lock })
        .catch(console.error)
    } else {
      wakeLockRef.current?.release().catch(console.error)
      wakeLockRef.current = null
    }

    return () => {
      wakeLockRef.current?.release().catch(console.error)
    }
  }, [status])
  ```

- 수정 대상 파일:
  - [x] `src/screens/TimerScreen.tsx` — Wake Lock 로직 추가
- ⚠ 참고/제약사항:
  - `navigator.wakeLock`은 Android Chrome/WebView 에서 지원 (Android 7.0+)
  - 앱이 백그라운드로 전환되면 Wake Lock이 자동 해제됨 — `visibilitychange` 이벤트로 복구 처리 권장
  - HTTPS 또는 네이티브 WebView 환경에서만 동작
- 테스트 필요: Yes — 타이머 실행 중 화면 꺼짐 없는지 확인

  **Step 3: Dexie DB 클래스 생성**

  파일: `src/db/focusDb.ts`
  ```ts
  import Dexie, { type Table } from 'dexie'

  export interface SessionMemo {
    sessionId: string   // FocusSession.id (PK)
    date: string        // YYYY-MM-DD (인덱스)
    memo: string
    createdAt: number   // timestamp
  }

  export class FocusDb extends Dexie {
    memos!: Table<SessionMemo, string>

    constructor() {
      super('FocusTimerDB')
      this.version(1).stores({
        memos: 'sessionId, date, createdAt',
      })
    }
  }

  export const db = new FocusDb()
  ```

  **Step 4: memoStore 구현**

  파일: `src/store/memoStore.ts`
  ```ts
  import { create } from 'zustand'
  import { db, type SessionMemo } from '../db/focusDb'

  interface MemoState {
    memos: SessionMemo[]
    loadAll: () => Promise<void>
    saveMemo: (sessionId: string, date: string, memo: string) => Promise<void>
    getMemoBySessionId: (sessionId: string) => SessionMemo | undefined
  }

  export const useMemoStore = create<MemoState>((set, get) => ({
    memos: [],

    loadAll: async () => {
      const all = await db.memos.orderBy('createdAt').reverse().toArray()
      set({ memos: all })
    },

    saveMemo: async (sessionId, date, memo) => {
      const entry: SessionMemo = { sessionId, date, memo, createdAt: Date.now() }
      await db.memos.put(entry)
      const all = await db.memos.orderBy('createdAt').reverse().toArray()
      set({ memos: all })
    },

    getMemoBySessionId: (sessionId) => {
      return get().memos.find((m) => m.sessionId === sessionId)
    },
  }))
  ```

  - Web Share API Level 2 (`navigator.share({ files })`)를 사용하면 이미지 파일도 공유 가능
  - 화면 캡처는 `html2canvas` 라이브러리로 DOM 요소를 `canvas`로 변환 → `canvas.toBlob()` → `File` 객체 생성

- 구현 내용:


  **Step 2: `src/utils/share.ts` 확장**

  ```ts
  export interface ShareScreenshotInput {
    elementRef: HTMLElement   // 캡처할 DOM 요소
    filename?: string         // 저장 파일명 (기본: focustimer-result.png)
    title?: string
    text?: string
  }

  export async function shareScreenshot(input: ShareScreenshotInput): Promise<ShareResult> {
    const html2canvas = (await import('html2canvas')).default
    const canvas = await html2canvas(input.elementRef, { backgroundColor: '#0f0f23' })

    return new Promise((resolve, reject) => {
      canvas.toBlob(async (blob) => {
        if (!blob) { reject(new Error('Screenshot failed')); return }

        const file = new File([blob], input.filename ?? 'focustimer-result.png', { type: 'image/png' })

        // Web Share API Level 2 (파일 공유 지원 여부 확인)
        if (navigator.canShare?.({ files: [file] })) {
          await navigator.share({ files: [file], title: input.title, text: input.text })
          resolve('shared')
          return
        }

        // Fallback: 이미지 다운로드
        const url = URL.createObjectURL(blob)
        const a = document.createElement('a')
        a.href = url
        a.download = input.filename ?? 'focustimer-result.png'
        a.click()
        URL.revokeObjectURL(url)
        resolve('copied')  // 'copied' 재사용 (다운로드 완료 의미)
      }, 'image/png')
    })
  }
  ```

  ```ts
  import type { SessionMemo } from '../db/focusDb'

  /** 단어 빈도 Top N 반환 (2자 이상, 불용어 제외) */
  export function getTopWords(memos: SessionMemo[], topN = 10): { word: string; count: number }[] {
    const stopWords = new Set(['그리고', '하지만', '그런데', '에서', '으로', '에게', '을', '를', '이', '가', '은', '는'])
    const freq = new Map<string, number>()

    for (const { memo } of memos) {
      for (const word of memo.split(/\s+/)) {
        const w = word.replace(/[^가-힣a-zA-Z0-9]/g, '').trim()
        if (w.length >= 2 && !stopWords.has(w)) {
          freq.set(w, (freq.get(w) ?? 0) + 1)
        }
      }
    }

    return [...freq.entries()]
      .sort((a, b) => b[1] - a[1])
      .slice(0, topN)
      .map(([word, count]) => ({ word, count }))
  }

  /** 시간대별(0~23시) 집중 세션 수 반환 */
  export function getHourlyDistribution(memos: SessionMemo[]): number[] {
    const hours = Array(24).fill(0) as number[]
    for (const { createdAt } of memos) {
      hours[new Date(createdAt).getHours()]++
    }
    return hours
  }
  ```

### ISSUE-054: 메모 입력 자동완성 — Dropdown 리스트 (최근·자주·입력 중 검색)

> 출처: brief.md — 메모 입력 시 최근·자주 사용한 내용 + 입력 중 내용 검색 결과를 DropdownList로 표시, 클릭 시 즉시 입력

- ⏱ 예상: AI 90분 / 시니어 2시간 | 🤖 Subagent: 없음

- 배경:
  - 동일하거나 유사한 작업 메모를 반복 입력하는 경우가 많음
  - textarea에 입력하는 즉시 과거 메모에서 매칭 결과를 드롭다운으로 표시 (search-as-you-type)
  - 입력이 없을 때는 최근 5건 + 자주 쓴 Top 5 를 드롭다운으로 제안
  - Fuse.js(이미 설치)를 활용한 유사도 검색

- 구현 내용:

  **`src/utils/memoSuggestions.ts` (신규)**

  ```ts
  import Fuse from 'fuse.js'
  import type { SessionMemo } from '../db/focusDb'

  /** 최근 N건 메모 텍스트 반환 (중복 제거) */
  export function getRecentMemos(memos: SessionMemo[], n = 5): string[] {
    const seen = new Set<string>()
    const result: string[] = []
    for (const { memo } of [...memos].sort((a, b) => b.createdAt - a.createdAt)) {
      const text = memo.trim()
      if (text && !seen.has(text)) { seen.add(text); result.push(text) }
      if (result.length >= n) break
    }
    return result
  }

  /** 빈도 Top N 메모 텍스트 반환 (중복 제거) */
  export function getFrequentMemos(memos: SessionMemo[], n = 5): string[] {
    const freq = new Map<string, number>()
    for (const { memo } of memos) {
      const text = memo.trim()
      if (text) freq.set(text, (freq.get(text) ?? 0) + 1)
    }
    return [...freq.entries()].sort((a, b) => b[1] - a[1]).slice(0, n).map(([t]) => t)
  }

  /** 입력 중인 텍스트로 과거 메모 유사도 검색 (Fuse.js) */
  export function searchMemos(memos: SessionMemo[], query: string, n = 8): string[] {
    if (!query.trim()) return []
    const fuse = new Fuse(memos, { keys: ['memo'], threshold: 0.4 })
    return fuse.search(query).slice(0, n).map((r) => r.item.memo)
  }
  ```
