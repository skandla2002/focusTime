### ISSUE-018: Android 필수 권한 및 Manifest 설정

> 출처: brief.md — Android 앱 출시를 위한 제약사항 조사 결과

- 구현 범위:
  - `android/app/src/main/AndroidManifest.xml` 에 아래 권한 추가:
    - `FOREGROUND_SERVICE` — Foreground Service 실행 허용
    - `FOREGROUND_SERVICE_SPECIAL_USE` — (Android 14+) 타이머 목적 명시
    - `POST_NOTIFICATIONS` — 알림 권한 (Android 13+)
    - `SCHEDULE_EXACT_ALARM` — 정확한 알람 (Android 12+)
    - `RECEIVE_BOOT_COMPLETED` — 재부팅 후 알림 복구

  - `AndroidManifest.xml` — AdMob App ID `<meta-data>` 태그 추가 (ISSUE-015 전제 조건)
  - 64-bit 아키텍처(`arm64-v8a`) 지원 확인 — Capacitor 기본 지원이나 빌드 검증 필요
- 전제 조건: ISSUE-014 완료
- 수정 대상 파일:
  - [ ] `android/app/src/main/AndroidManifest.xml`
  - [ ] `android/app/build.gradle`

### ISSUE-021: 하단 탭 위 배너 광고 레이아웃 구현

> 출처: brief.md — 하단 탭에 광고 영역이 없어 수익 창출 불가

- 배경:
  - `BannerAd.tsx` 컴포넌트가 이미 존재하나 `App.tsx`에 삽입되지 않아 화면에 노출되지 않음
  - `App.tsx` 현재 구조: `{화면 컨텐츠}` → `<Navigation />` → `<InterstitialAd />`
  - 배너를 삽입할 경우 컨텐츠 영역이 배너+탭바 높이만큼 가려지는 레이아웃 문제 발생
- 구현 범위:
  - `App.tsx` — `<Navigation />` 바로 위에 `<BannerAd />` 삽입

    ```jsx
    <div class="app">
      {화면 컨텐츠}
      <BannerAd />       {/* 추가 (탭바 바로 위) */}
      <Navigation />
      <InterstitialAd />
    </div>
    ```

  - `App.css` — `.app` 레이아웃 조정:
    - 화면 컨텐츠 영역의 `padding-bottom` 을 `탭바 높이(60px) + 배너 높이(60px)`로 확보
    - `.banner-ad` 고정 높이 60px, `position: fixed`, `bottom: 60px` (탭바 위)
    - 탭바(`.nav`)는 `position: fixed`, `bottom: 0` 유지
  - `BannerAd.tsx` — 현재 placeholder 유지 (실제 SDK 연동은 ISSUE-015에서 처리)
- 수정 대상 파일:
  - [ ] `src/App.tsx`
  - [ ] `src/App.css`
- ⚠ 참고/제약사항:
  - 배너는 화면 전환과 관계없이 항상 노출 (타이머 화면 포함)
  - 실제 AdMob 배너 표준 크기: `BANNER` 320×50dp, `ADAPTIVE_BANNER` 전체 폭×60dp — CSS 높이를 60px로 통일
  - 웹 환경(개발)에서는 placeholder 텍스트 표시, 네이티브(ISSUE-015) 이후 실광고 전환
